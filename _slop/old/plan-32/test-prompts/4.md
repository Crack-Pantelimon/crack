# Test 4 — Rust refactor primitive: replace a symbol body

Paste into the chat:

---

Use the **serena** MCP tools only.

1. Call `find_symbol` for `spawn_root_map_tiles` (a Rust `pub fn`) with the body
   included, and show me the current body.
2. Add a single leading log line to the top of that function using
   `replace_symbol_body` (or `insert_after_symbol` at the opening brace) — e.g.
   `info!("spawn_root_map_tiles: serena refactor test");` — without touching the rest
   of the body. Show me the diff serena produced.
3. Revert the change (put the original body back with `replace_symbol_body`) and
   confirm the file is byte-identical to before.

Do the edit through serena's symbol-level tools, not a raw text edit.

---

**What this checks:** the symbol-level *editing* primitives (`replace_symbol_body` /
`insert_after_symbol`) that operate on rust-analyzer's understanding of where a
function begins and ends — the building block for larger automated refactors.
If needed, `git checkout -- .` restores the original.
