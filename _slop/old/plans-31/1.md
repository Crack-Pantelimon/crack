# Replace the docs-mcp RAG backend with zilliztech/claude-context + Milvus

## Context

The current self-hosted RAG stack (design & code under `_slop/plans-30-rag/`, wired in
`_docker/` and `.pi/crack/server/`) is built on **arabold/docs-mcp-server** — a Node tool
with its own **SQLite** vector store and **Ollama** embeddings. We're dissatisfied with it
(notably: it has *no* `.gitignore` awareness, so it scraped and embedded generated data —
see `_slop/plans-30-rag/3_report.md` — forcing a bespoke `_docs_mcp_gitignore_patterns.py`
workaround; and its store/search quality is weak vs. AST-aware tools).

We want to swap the **implementor** for the industry-standard **zilliztech/claude-context**
(12.2k★): AST-aware chunking, hybrid BM25 + dense search, incremental Merkle re-index,
Milvus vector store, and **native `.gitignore` support**. All user-facing functionality
stays — the `/rag` search pages, first-hop injection, and the in-agent search MCP tool —
only the backend changes. Now that we have `_docker/docker-compose.yml`, we add the Milvus
standalone stack there.

## Locked decisions (confirmed with user)

- **Embeddings: keep the local `ollama` container.** Milvus only stores/searches vectors;
  claude-context still needs an embedder. We point claude-context's Ollama provider at the
  existing `ollama` service (`nomic-embed-text`). **Ollama is NOT removed** — it's still
  needed, just now serving claude-context instead of docs-mcp. No API keys, no external network.
- **Index scope: repo only** (`/workspace`, gitignore-aware). Drop the old Rust-dep-source
  and Python-site-packages indexing. (Dep-source search is a possible later follow-up.)

## The single integration seam

Everything funnels through **`search_docs()` in
[.pi/crack/server/src/crack_server/rag.py](.pi/crack/server/src/crack_server/rag.py#L69-L134)**,
which returns `list[{score, source, snippet}]`. The `/rag` pages
([routes_rag.py](.pi/crack/server/src/crack_server/routes_rag.py)), first-hop injection
([rag_inject.py](.pi/crack/server/src/crack_server/rag_inject.py)), and all their tests
consume only that shape. **We re-point the body of `search_docs`; the contract is unchanged**,
so `routes_rag.py`, `rag_inject.py`, `settings.rag_config()`, and `harness/rag.json` need
essentially no change beyond one header label.

## New architecture / data flow

- **Vector store:** self-hosted **Milvus standalone** = 3 containers (`milvus-standalone`,
  `milvus-etcd`, `milvus-minio`) in `docker-compose.yml`, reachable at
  `milvus-standalone:19530` on `crack-net`.
- **Embedder:** existing `ollama` container, `nomic-embed-text` (dim 768).
- **Indexer + query:** a tiny Node package under
  `/crack-harness-data/tools/claude-context/` using **`@zilliz/claude-context-core`**
  (`Context` → `indexCodebase()` / `semanticSearch()`; confirmed usable as a standalone
  script, result shape `{content, relativePath, startLine, endLine, language, score}`).
- **In-agent MCP tool:** claude-context ships a **stdio** MCP server
  (`@zilliz/claude-context-mcp`) exposing `search_code` / `index_codebase` — wired directly
  in `.mcp.json` (no `supergateway`/SSE bridge, no `:9933` HTTP server needed anymore).
- All three consumers (boot indexer, Python `search_docs`, agent MCP) point at the **same**
  `/workspace` codebase + same Milvus + same embedding config, so they share one collection.

## Changes by area

### 1. `_docker/docker-compose.yml` — add Milvus, keep ollama, drop the :9933 port
- Add 3 services (standard Milvus standalone compose): `milvus-etcd`
  (`quay.io/coreos/etcd`), `milvus-minio` (`minio/minio`), `milvus-standalone`
  (`milvusdb/milvus:v2.5.x`, `command: milvus run standalone`, `ETCD_ENDPOINTS`,
  `MINIO_ADDRESS`, expose `19530`, `depends_on` etcd+minio, healthcheck on `:9091/healthz`).
- Add named volumes: `milvus-etcd-data`, `milvus-minio-data`, `milvus-data`.
- Keep the `ollama` service and `ollama-data` volume **unchanged**.
- `crack-dev`: `depends_on: [ollama, milvus-standalone]`; **remove** the `127.0.0.1:9933:9933`
  port line (no more docs-mcp HTTP server).

### 2. New container scripts (replace the four `_docs_mcp_*` files)
Create, mirroring the existing helper structure:
- **`_docker/_claude_context_setup.sh`** (replaces `_docs_mcp_setup.sh`): defines the shared
  env — `MILVUS_ADDRESS=milvus-standalone:19530`, `EMBEDDING_PROVIDER=Ollama`,
  `OLLAMA_HOST=http://ollama:11434`, `EMBEDDING_MODEL=nomic-embed-text`,
  `EMBEDDING_DIMENSION=768` (skips runtime probe), `NODE_ENV=production`; a
  `claude_context_ensure_built()` that `npm install`s `@zilliz/claude-context-core` into
  `/crack-harness-data/tools/claude-context/` and drops in `index.mjs` + `search.mjs`
  (see §3); `claude_context_wait_milvus()` (poll `:9091/healthz`) and reuse the existing
  Ollama readiness + `nomic-embed-text` pull logic.
- **`_docker/_claude_context_index.sh`** (replaces `_docs_mcp_index.sh` + deletes the whole
  gitignore-glob workaround): keep the fingerprint/stamp skip (git HEAD + lockfiles), gate on
  Milvus + ollama-model ready, then a single `node index.mjs /workspace`. claude-context is
  **gitignore-aware natively**, so `_docs_mcp_gitignore_patterns.py` is deleted entirely; use
  `CUSTOM_IGNORE_PATTERNS` only for the few extra dirs (`target`, `node_modules`, `_slop`,
  `.playwright-mcp`) not already covered by tracked `.gitignore`s.
- **Delete** `_docker/_docs_mcp_setup.sh`, `_docker/_docs_mcp_index.sh`,
  `_docker/_docs_mcp_lazy.sh`, `_docker/_docs_mcp_gitignore_patterns.py` (all currently
  staged/new — being replaced, not reverting committed history).

### 3. `tools/claude-context/` Node package (written by `ensure_built`)
- `package.json` depending on `@zilliz/claude-context-core` (pin a version).
- **`index.mjs`**: instantiate `Context` with `MilvusVectorDatabase({address})` +
  `OllamaEmbedding`/OpenAI-compatible embedding from env; `await context.indexCodebase(argv.path)`.
- **`search.mjs`**: same `Context`; `semanticSearch('/workspace', query, limit)`; print the
  results as a JSON array to stdout (the shape `search_docs` will parse). Fail-open friendly
  (non-zero exit / empty array on any error).

### 4. `_docker/_cont_start.sh` — swap the docs-mcp boot block (lines ~88-102)
Replace with a backgrounded claude-context block: source `_claude_context_setup.sh`,
`claude_context_ensure_built`, wait for Milvus + ollama model, then
`bash _claude_context_index.sh` (stamped, incremental). **No respawn'd HTTP server** — the
`:9933` server and `MCP_DOCS_PORT` go away. Keep it backgrounded so `crack-server` doesn't
block on the cold index.

### 5. `.mcp.json` — repoint the agent tool
Replace the `docs-search` server (which runs `_docs_mcp_lazy.sh` + supergateway) with a
`code-search` server: `command: npx`, `args: [-y, @zilliz/claude-context-mcp@<pin>]`, and an
`env` block carrying `MILVUS_ADDRESS`, `EMBEDDING_PROVIDER=Ollama`, `OLLAMA_HOST`,
`EMBEDDING_MODEL`, `EMBEDDING_DIMENSION`. (`.mcp.json` is copied into sandboxes by
`_docker/_sandbox_common.sh`, so agents inherit it automatically.) Also update the
`docs-search` reference comment in `_cont_start.sh:26`.

### 6. `.pi/crack/server/src/crack_server/rag.py` — re-point `search_docs`
- Replace the `DOCS_MCP_*`/`OLLAMA` module constants with claude-context ones:
  `CLAUDE_CONTEXT_DIR=/crack-harness-data/tools/claude-context`, `CODEBASE_PATH=/workspace`,
  `MILVUS_ADDRESS`, plus the embedding env (Ollama). Keep `RAG_SEARCH_TIMEOUT`.
- `_docs_mcp_env()` → `_claude_context_env()`: export `MILVUS_ADDRESS`,
  `EMBEDDING_PROVIDER=Ollama`, `OLLAMA_HOST`, `EMBEDDING_MODEL`, `EMBEDDING_DIMENSION`.
- `dist_path()`/`available()`: point at `.../claude-context/search.mjs` existing (drop the
  SQLite-store check; optionally treat "script present" as available).
- **`search_docs()`**: keep the exact async signature and fail-open behavior; change only the
  subprocess to `node <CLAUDE_CONTEXT_DIR>/search.mjs <query> --limit <n>` (drop the
  `library` positional — claude-context searches a codebase path, not a named library; keep
  the `library` kwarg accepted-but-ignored for call-site compatibility).
- **`normalize_hit()`**: map claude-context results → the unchanged `{score, source, snippet}`
  contract: `source = relativePath` (append `:startLine-endLine` for a clickable-ish ref),
  `snippet = content`, `score = score`.

### 7. `.pi/crack/server/src/crack_server/routes_rag.py` — header label only

backend-agnostic label (e.g. `rag.CODEBASE_LABEL = "crack-repo"`). No route/UX change —
`/rag` and `/rag/search` keep working via `search_docs`.

### 8. Stray-data cleanup (old docs-mcp / ollama volumes)
