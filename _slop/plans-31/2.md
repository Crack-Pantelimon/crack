# Review-fix the claude-context/Milvus swap, switch to a lighter embed model, bring up the stack

## Context

A weaker model already implemented the docs-mcp → **zilliztech/claude-context + Milvus**
swap (see `_slop/plans-31-milvus`). The Milvus standalone stack, the `_claude_context_*`
boot scripts, the `tools/claude-context/` Node templates, the re-pointed `search_docs`
seam, the `.mcp.json` `code-search` server, and new tests all exist. My review confirms the
claude-context-core API usage is **correct** (`hasIndex`/`reindexByChange`/`indexCodebase`/
`semanticSearch`, `OllamaEmbedding({model,host,dimension})`, `Context({..., customIgnorePatterns})`
all verified against the v0.1.x source; `semanticSearch` returns `{content, relativePath,
startLine, endLine, language, score}` which matches `normalize_hit`).

This plan: apply three fixes, switch the embedding model to a lighter one (per user), then
build + bring up the stack and clear stale docs-mcp data.

## Fixes to apply

### 1. `_docker/claude-context/search.mjs` — don't let the 0.5 threshold eat all hits
`semanticSearch(path, query, topK, threshold=0.5)` currently gets only 3 args, so the
default **0.5** filters out real hits (the injection uses `first_hop_min_score=0.02`; the
`/rag` page must show ranked hits regardless of absolute score). Read
`RAG_SEARCH_THRESHOLD` (default `"0"`), parse to float, and pass it as the 4th arg. Downstream
`min_score` filtering in `rag_inject.py` still governs injection.

### 2. `_docker/docker-compose.yml` — fix the minio healthcheck image
`minio/minio:RELEASE.2024-12-18T13-15-44Z` removed `curl`, so its
`curl -f .../minio/health/live` healthcheck can never pass. Pin to the version the official
Milvus v2.5 standalone compose uses — **`minio/minio:RELEASE.2023-03-20T20-16-18Z`** (ships
curl). Also bump etcd `v3.5.16` → **`v3.5.18`** to match official. (milvus `v2.5.10` is fine.)

### 3. Switch the embedding model: `nomic-embed-text` (768) → `all-minilm` (384)
Smaller/faster Ollama model (all-MiniLM-L6-v2, 22M params, 384-dim vs nomic's 137M/768-dim).
Update the default + add a switch-back comment in **all four** places that must agree (the
embedding model and dimension are baked into the Milvus collection, so they must match across
indexer, query CLI, and agent MCP):
- `_docker/_claude_context_setup.sh` — `EMBEDDING_MODEL` / `EMBEDDING_DIMENSION` defaults.
- `.pi/crack/server/src/crack_server/rag.py` — `EMBEDDING_MODEL` / `EMBEDDING_DIMENSION` defaults.
- `.mcp.json` — `code-search` server `env` (`EMBEDDING_MODEL`, `EMBEDDING_DIMENSION`).
- (fingerprint in `_claude_context_index.sh` auto-changes via the version/env, forcing reindex — no edit needed, but the model/dim are part of `claude_context_env`; confirm the fingerprint includes enough to re-trigger — if not, add `EMBEDDING_MODEL` to `index_fingerprint`.)

Comment to drop at each site (verbatim intent):
```
# Embedding model for indexing + query. Kept small/fast (all-minilm, 384-dim).
# For higher retrieval quality switch BOTH the model and dimension together and
# re-index (the dim is baked into the Milvus collection):
#   nomic-embed-text  -> EMBEDDING_DIMENSION=768   (stronger, ~137M)
#   mxbai-embed-large -> EMBEDDING_DIMENSION=1024  (strongest common ollama embed)
# `ollama pull <model>` happens automatically at boot (claude_context_ensure_embed_model).
```
